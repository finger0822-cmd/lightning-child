# デプロイ反映されない原因調査チェックリスト

## 確認事項

### 1. サーバー上のファイル確認
- [ ] FTPでサーバー上の `page-ouchi-support.php` を直接確認
- [ ] 19-20行目: `月額10,000円で、暮らしに「安心の習慣」を。` か確認
- [ ] 118行目: `月額10,000円（税込）／月2回訪問（各60分）` か確認
- [ ] ファイルの更新日時が最新か確認

### 2. WordPress設定確認
- [ ] 固定ページ（ID: 491）で「ページ属性」→「テンプレート」が「おうち安心サポート」になっているか
- [ ] 別のテンプレートが選択されていないか確認
- [ ] 固定ページのスラッグが `/support/` になっているか確認

### 3. キャッシュ確認
- [ ] WordPress管理画面でキャッシュプラグインのキャッシュをクリア
- [ ] ブラウザのキャッシュを完全にクリア（`Ctrl + Shift + Delete`）
- [ ] シークレット/プライベートモードで確認
- [ ] サーバーのOPcacheをクリア（可能であれば）

### 4. テンプレート読み込み確認
以下のコードを一時的に追加して、どのファイルが読み込まれているか確認：

```php
<!-- DEBUG: このファイルが読み込まれています: <?php echo __FILE__; ?> -->
<!-- DEBUG: テンプレート: <?php echo basename(__FILE__); ?> -->
```

### 5. 可能性のある原因
1. **OPcache**: PHPのOPcacheが古いファイルをキャッシュしている
2. **オブジェクトキャッシュ**: WordPressのオブジェクトキャッシュ（Redis、Memcachedなど）
3. **ページキャッシュ**: プラグイン（WP Super Cache、W3 Total Cacheなど）
4. **CDN/プロキシ**: CloudflareなどのCDNが古いコンテンツをキャッシュ
5. **別のテンプレートファイル**: 親テーマや別の場所に同名ファイルがある

### 6. 確認方法
- ブラウザの開発者ツール（F12）→ ネットワークタブ → レスポンスヘッダーを確認
- サーバー上のファイルを直接ブラウザで開いてみる（認証が必要な場合は一時的に）
- ファイルに一時的にコメントを追加して、それが表示されるか確認
