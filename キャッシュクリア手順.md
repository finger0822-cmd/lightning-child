# キャッシュクリア手順 - 料金反映されない問題の解決

## 問題
サーバー上のファイルは10,000円に更新済みだが、サイトでは9,800円が表示される

## 原因
**キャッシュ**が原因の可能性が非常に高いです。

## 解決手順（優先順位順）

### 1. ブラウザのキャッシュを完全にクリア（最重要）
1. **Windows:**
   - `Ctrl + Shift + Delete` を押す
   - 「キャッシュされた画像とファイル」にチェック
   - 「全期間」を選択
   - 「データを削除」をクリック
2. **シークレット/プライベートモード**でページを開く
   - Chrome: `Ctrl + Shift + N`
   - Edge: `Ctrl + Shift + P`
   - Firefox: `Ctrl + Shift + P`
3. **強制リロード:**
   - `Ctrl + F5` または `Ctrl + Shift + R`

### 2. WordPress管理画面でのキャッシュクリア
1. WordPress管理画面にログイン
2. 使用中のキャッシュプラグインを確認
   - **WP Super Cache**: 設定 → WP Super Cache → 「キャッシュを削除」
   - **W3 Total Cache**: Performance → Dashboard → 「すべてのキャッシュをクリア」
   - **LiteSpeed Cache**: LiteSpeed Cache → ツール → 「全キャッシュを削除」
   - **WP Rocket**: WP Rocket → 「キャッシュを削除」
3. プラグインがない場合も、WordPressの標準キャッシュをクリア：
   - 外観 → カスタマイズ → キャッシュクリアボタンを探す

### 3. サーバー側のキャッシュをクリア

#### PHP OPcacheのクリア
1. サーバーのコントロールパネルにログイン
2. 「PHP設定」または「OPcache」を探す
3. 「OPcacheをクリア」または「リセット」をクリック

または、一時的に `.htaccess` に追加（その後削除）：
```apache
php_value opcache.enable 0
```

#### オブジェクトキャッシュのクリア（Redis/Memcached使用時）
1. サーバーのコントロールパネルでRedis/Memcachedの管理画面を開く
2. 「Flush」または「すべて削除」を実行

### 4. CDN/プロキシのキャッシュをクリア
Cloudflareなどを使用している場合：
1. Cloudflareダッシュボードにログイン
2. 「キャッシュ」→「キャッシュのパージ」→「すべてをパージ」

### 5. ファイルの更新日時を確認
FTPクライアントで以下を確認：
```
/public_html/suzukireinetsu.com/wp-content/themes/lightning-child/page-ouchi-support.php
```
- ファイルの更新日時が最新（数分以内）になっているか確認
- 更新されていない場合は、手動でFTPアップロード

### 6. WordPressテンプレートの確認
1. WordPress管理画面 → 固定ページ → 該当ページを編集
2. 「ページ属性」→「テンプレート」が「おうち安心サポート」になっているか確認
3. 正しいテンプレートが選択されているか確認

### 7. 一時的な確認方法
ページのソースを確認（`Ctrl + U`）：
- HTMLソースに `<!-- DEBUG: ファイルバージョン 2.0` が含まれていれば、新しいファイルが読み込まれている
- 含まれていなければ、古いファイルが読み込まれている可能性

## 確認方法
1. 上記の手順をすべて実行
2. ブラウザを完全に閉じて、再度開く
3. シークレットモードで `/support/` ページを開く
4. ページのソースを確認（`Ctrl + U`）→ `10,000円` を検索（`Ctrl + F`）

## それでも解決しない場合
1. サーバー上のファイルを直接確認（FTP）
2. ファイルのパーミッションを確認（644推奨）
3. WordPressのデバッグモードを有効にして、エラーを確認
