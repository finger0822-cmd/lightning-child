# リポジトリ名変更手順

## 現在のリポジトリ
- リポジトリ名: `businesspress-child`
- URL: https://github.com/finger0822-cmd/businesspress-child

## 変更後のリポジトリ
- リポジトリ名: `lightning-child`
- URL: https://github.com/finger0822-cmd/lightning-child

## 変更手順

### ステップ1: GitHubでリポジトリ名を変更
1. GitHubのリポジトリページを開く: https://github.com/finger0822-cmd/businesspress-child
2. 右上の「Settings」（設定）タブをクリック
3. ページの一番下までスクロールして「Danger Zone」セクションを見つける
4. 「Change repository name」をクリック
5. 新しい名前を入力: `lightning-child`
6. 「I understand, change repository name」をクリックして確認

### ステップ2: ローカルのリモートURLを更新
GitHubで名前を変更した後、以下のコマンドを実行：

```bash
cd C:\Users\taker\lightning-child
git remote set-url origin https://github.com/finger0822-cmd/lightning-child.git
git remote -v
```

### ステップ3: 動作確認
```bash
git fetch origin
git status
```

## 注意事項
- GitHubでリポジトリ名を変更すると、古いURLは自動的に新しいURLにリダイレクトされます
- 既存のクローンやフォークは自動的には更新されません（手動で更新が必要）
- GitHub Actionsなどの設定も自動的には更新されません（手動で確認が必要）

## GitHub Actionsの設定
リポジトリ名を変更しても、GitHub Actionsのワークフローはそのまま動作します。
ただし、ワークフローのURLは変更されます。
