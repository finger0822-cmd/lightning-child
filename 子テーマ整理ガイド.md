# 子テーマが2つある場合の整理方法

## 現在の状況

- ✅ 親テーマ（Lightning）はインストールされている
- ⚠️ 子テーマが2つ表示されている

## 考えられる原因

サーバー上に以下の2つの子テーマディレクトリが存在している可能性があります：

1. `/wp-content/themes/businesspress-child/`（古い・削除対象）
2. `/wp-content/themes/lightning-child/`（新しい・使用中）

## 解決方法

### 方法1: WordPress管理画面から削除（推奨）

1. **WordPress管理画面にログイン**

2. **「外観」→「テーマ」を開く**

3. **表示されている2つの子テーマを確認**
   - テーマ名を確認して、どちらが正しいか判断
   - 「Lightning Child」が正しい子テーマ

4. **不要な子テーマを削除**
   - 削除したいテーマの上にマウスを置く
   - 「テーマの詳細」をクリック
   - 左下の「削除」ボタンをクリック

5. **正しい子テーマ（Lightning Child）を有効化**

### 方法2: FTP/SFTPで手動削除

1. **FTP/SFTPでサーバーに接続**

2. **以下のディレクトリ構造を確認**
   ```
   /wp-content/themes/
   ├── lightning/              ← 親テーマ（残す）
   ├── lightning-child/        ← 新しい子テーマ（残す）
   └── businesspress-child/    ← 古い子テーマ（削除）
   ```

3. **古い `businesspress-child` ディレクトリを削除**

### どちらの子テーマを残すべきか

**残すべき子テーマ:**
- テーマ名：`Lightning Child`
- ディレクトリ：`lightning-child`
- `Template: lightning` と設定されている

**削除すべき子テーマ:**
- テーマ名：`BusinessPress Child` または古い名前
- ディレクトリ：`businesspress-child`
- `Template: businesspress` など、Lightning以外が設定されている

## 親テーマが表示されない件について

**これは正常な動作です。**

- 親テーマはインストールされているが、子テーマがある場合は表示されないことがあります
- 親テーマを有効化する必要はありません
- 子テーマを有効化すると、親テーマは自動的に読み込まれます

## 確認事項

整理後、以下を確認してください：

1. ✅ 子テーマが1つだけ表示される（Lightning Child）
2. ✅ Lightning Childテーマが正常に有効化できる
3. ✅ サイトが正常に表示される

## 重要なポイント

- **親テーマは有効化する必要がありません**
- **子テーマ（Lightning Child）のみを有効化してください**
- **不要な子テーマは削除して整理してください**
